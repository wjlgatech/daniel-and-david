# Safety & Trust 🛡️

Children read, use, and contribute to this repo. So safety is not a disclaimer bolted on at
the end — **it is part of the product.** This folder is the visible governance layer that has
to exist *before* we build accounts, profiles, uploads, matching, or messaging.

<p align="center"><img src="../assets/safety-layer.svg" alt="Safety is part of the product. Core rules that bind humans and AI: no child personal data (no full names, location, school, schedule); no public child profiles, no private adult-child messages, no unmediated matching; every public child contribution is reviewed before it appears; AI never replaces the child's own thinking and never collects their data by default; and when a safety rule and a feature conflict, the rule wins and the feature waits." width="100%"></p>

> follow them exactly — see [`AGENTS.md`](../../AGENTS.md). When a rule and a feature conflict,
> the rule wins and the feature waits.
