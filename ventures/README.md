# Ventures

Real businesses, designed in the open. This is where principles meet payroll — where the boys
(and contributors worldwide) turn ideas into things people actually pay for.

> 🛠️ **Ventures are the graduation, not the starting line.** A child learns the moves first on
> the four-week **[Builder Loop](../docs/builder-loop/)** (5 fast build-show-learn cycles —
> fast, frequent, failing-forward), then points that same engine at a real venture.

## The studio model

Each venture is a self-contained folder with:

- a **`README.md`** — what it is, the offer, current build status,
- a **`SPEC.md`** — the full operating spec (economics, compliance, go/no-go gates, PRDs),
- and, as it grows, an **`app/`** and whatever else it needs to run.

We run ventures like an [AI-native company](../docs/principles/ai-native-company.md): small
human team, AI teammates doing the heavy lifting, everything measured, docs shipped with the
work, and the [values](../docs/principles/values.md) held without exception.

## Starting a new venture

Have an idea? The bar is simple: **does it solve a real problem for real people, legally and
honestly?** If yes:

1. Open a [Discussion](https://github.com/wjlgatech/daniel-and-david/discussions) to pitch it.
2. If it gets traction, add `ventures/your-venture/` with a `README.md` and a `SPEC.md`.
3. Follow the same shape as venture #1: clear offer, honest economics, compliance gates,
   and a buildable plan.

Every venture must pass the [four-question test](../docs/principles/values.md#the-test-for-any-decision):
is it true, is it legal, does it serve, would you be proud to explain it to your kids and to
God?
